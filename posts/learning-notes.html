學習筆記：靜態網站部署

2026 年 5 月 17 日 · 標籤： 技術 GitHub

這篇文章記錄了我使用 GitHub Pages 部署靜態網站的完整步驟。

什麼是 GitHub Pages？

GitHub Pages 是 GitHub 提供的免費靜態網站託管服務。你只需要將 HTML、CSS、JavaScript 檔案推送到 GitHub 儲存庫，就能獲得一個公開的網站。

部署步驟

1. 建立 GitHub 儲存庫

在 GitHub 上建立一個新的儲存庫。如果你希望網址是 claudia-fox.github.io，就將儲存庫命名為 claudia-fox.github.io。

2. 推送程式碼

將你的網站檔案推送到儲存庫的 main 分支：

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/claudia-fox/claudia-fox.github.io.git
git push -u origin main

3. 啟用 GitHub Pages

進入儲存庫的 Settings → Pages，選擇部署來源為 main 分支的根目錄，儲存後等待幾分鐘即可。

注意事項

  • 確保根目錄有 index.html 檔案
  • 圖片和資源的路徑要使用相對路徑
  • 自訂網域需要額外設定 CNAME 檔案
  • 更新內容只需推送新的 commit 即可自動部署
GitHub Pages 適合個人部落格、作品集、專案文件等靜態內容的託管。

就是這麼簡單！如果你也想建立自己的部落格，現在就動手試試吧。
